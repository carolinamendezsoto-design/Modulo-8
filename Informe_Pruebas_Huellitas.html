🐾 HUELLITAS DE AMOR
INFORME OFICIAL DE PRUEBAS END-TO-END

Módulo 8 - Proyecto Final Integrador

1. Interfaz Principal, UI y Modificaciones (Login)

Muestra oficial de la página de acceso principal. Se comprueba gráficamente el diseño de UI Glassmorphism, el renderizado de cristales y la fijación responsiva de la Leyenda Técnica exigida para la Comuna de Puente Alto, superando el overlapping de medios.

2. Registro Biométrico y Validación Multiusuario

Demostración del registro seguro de entrada. En esta instancia el sistema de autenticación captura los roles asimétricos (Adoptante, Rescatista, Admin). Las contraseñas viajan bajo capas de bcrypt y se enjaulan con firma JSON Web Token antes de permitir el ingreso.

3. Módulo Rescatistas: Ingestión Multimedia (Crear Mascota)

Vista del Dashboard Autenticado para perfil Rescatista. Al crear la mascota, se validan los mimetypes en el servidor empleando el Middleware de Multer (JPG, WEBP, PNG; límite absoluto 5MB). Al subirse sin fallos, la mascota queda ligada algorítmicamente y asociada mediante FK en PostgreSQL.

4. Arquitectura Validada y Comprobaciones Transaccionales

Las pruebas mecánicas y de usabilidad concluyeron satisfactoriamente los 4 flujos exigidos. A continuación se evidencia el Catálogo de Adopciones Renderizando la mascota de prueba ("Mirana") recuperada limpiamente desde la base de datos PostgreSQL.

  • ✅ Login e Ingreso Asíncrono: Comprobación de seguridad API y validación de tokens en LocalStorage bloqueando rebotes de intrusos.
  • ✅ Roles (Admin, Rescatista, Adoptante): El control estricto redirige impecablemente hacia admin.html, rescatista.html o mascotas.html.
  • ✅ Comprobación de Adopción Total: Las solicitudes de adopción (N:M) quedan incrustadas exitosamente en la sub-tabla relacional (Mascota <-> Adoptante), disparando llamadas a la matriz del repositorio Notificacion.
  • ✅ Pestañas Interactivas: El total de vistas HTML (Incluyendo Postulantes y Campana de Notificaciones) transita sin mezclar solicitudes entre perfiles asíncronos.
